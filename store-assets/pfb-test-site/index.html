Jot for Windows — PFB Gateway Test

Run this on an in-network Sony Windows machine (on the VPN). Takes about 5 minutes. Two ways: the app itself (recommended), or raw commands to isolate a problem.

✅ Goal: confirm sign-in works and a Claude and a GPT model each return a reply. If both do, the Windows build is verified end-to-end.

A Test inside the Jot app (recommended)

  1. Open Jot → Settings → AI.
  2. Set Provider to PFB.
  3. If you see “Install sign-in helper”, tap it once (or run the command in section B step 1).
  4. Tap “Sign in to PFB” → complete the Okta login in the browser that opens.
  5. You should see Signed in as … · expires in ~24h.
  6. Pick a Model, then tap Test connection → expect “Connected to PFB successfully.”
  7. Real check: select some text in any app, trigger Rewrite (e.g. “Make shorter”), and confirm it comes back transformed.
Try it once with a Claude model and once with a GPT-5.6 model (switch in the Model dropdown) — that exercises both request shapes.

B Raw commands (only if the app fails — isolates auth/endpoint)

Open PowerShell and run these in order.

1. Install the sign-in helper (one time)

Invoke-WebRequest -Uri "https://download.ai.studios.playstation.com/ai-gateway-cli/binaries/gimme-ai-creds-windows-amd64.exe" -OutFile "$env:LOCALAPPDATA\gimme-ai-creds.exe"

2. Sign in & capture the token (opens Okta in the browser)

$env:TOKEN = (& "$env:LOCALAPPDATA\gimme-ai-creds.exe" --org pfb --quiet --no-clipboard).Trim()
$env:TOKEN.Substring(0,6)

That last line should print eyJ... — a clean JWT. If it’s blank or has extra text, the CLI’s stdout isn’t clean (that’s the one Windows-specific thing to confirm).

3. GPT-5.6 Luna — reasoning_effort:"none", no temperature, no token limit

@'
{"model":"gpt-5.6-luna","messages":[{"role":"user","content":"Say hello in 5 words."}],"reasoning_effort":"none"}
'@ | Set-Content "$env:TEMP\pfb-gpt.json" -Encoding ascii
curl.exe -s -X POST "https://ai-gateway.dspprod.bis.sie.sony.com/pfb/common/v1/chat/completions" -H "Authorization: Bearer $env:TOKEN" -H "Content-Type: application/json" -d "@$env:TEMP\pfb-gpt.json"

4. Claude Sonnet 5 — temperature, no reasoning_effort, no token limit

@'
{"model":"global.anthropic.claude-sonnet-5","messages":[{"role":"user","content":"Say hello in 5 words."}],"temperature":0.3}
'@ | Set-Content "$env:TEMP\pfb-claude.json" -Encoding ascii
curl.exe -s -X POST "https://ai-gateway.dspprod.bis.sie.sony.com/pfb/common/v1/chat/completions" -H "Authorization: Bearer $env:TOKEN" -H "Content-Type: application/json" -d "@$env:TEMP\pfb-claude.json"

Both should return a JSON reply containing choices[0].message.content with a short greeting.
