# Jot for Windows — UX/UI Audit (2026-07-20)

Agent-generated audit by a Windows UX/UI reviewer. Every finding cites a `file:line` the reviewer read.
Deliberately-gated (not-yet-real) features are listed at the end and excluded from the bug lists.

Scope: setup wizard, shell/nav, Recents + detail, Settings (basic + advanced), rewrite prompt picker +
Prompts page, status pill/captions, donations + nudge, About, Help, Shortcuts, tray/hotkeys/notifications.

## P1 — Contradictions & real UX bugs

**1. Wizard claims the speech model is already installed; Settings says it may not be.**
`SetupWizardWindow.xaml:56-58` renders a hardcoded green check + "On-device model ready (≈600 MB)" on the
Language step, with no binding to actual install state. The real install flow lives in Settings:
`SettingsPage.xaml:185-198` shows "Not installed (~0.67 GB)" + Download, driven by
`SettingsViewModel.cs:255-262` (`IsModelInstalled`). A fresh user is told they're "ready" while the 0.67 GB
model may be undownloaded. Two defects: the claim is unconditional, and the size disagrees with itself
(≈600 MB vs ~0.67 GB). *Fix:* bind the wizard status/check to `IsModelInstalled`; show a real
download/progress affordance (or "downloads on first use"); use one size string everywhere.

**2. "Launch at login" description says "in the tray," but the app opens the main window.**
`SettingsPage.xaml:154` — "Start Jot in the tray when you sign in." The registry entry is written with no
args (`SettingsViewModel.cs:404`), so login launch hits the no-arg path in `App.xaml.cs:322-324` →
`ShowMainWindow()` (intentional). Behavior fine; copy is false. *Fix:* "Open Jot when you sign in," or truly
boot to tray.

**3. Help's "Rewrite" description documents a different feature than the Rewrite shortcut runs.**
`HelpPage.xaml:21-22` says press Rewrite and "speak an instruction" — that's the *voice*-rewrite flow
(`ToggleVoiceRewrite`, `App.xaml.cs:2099`). The Rewrite shortcut actually opens a *typed* prompt picker
(`App.xaml.cs:2098` `BeginRewrite(OpenRewritePicker)`), which `ShortcutsPage.xaml:27-29` correctly describes.
Help and the shipped shortcut contradict. *Fix:* rewrite the Help card for the picker; document voice-rewrite
as its own shortcut.

**4. Finishing (or skipping) the wizard force-enables Advanced features, defeating the basic/advanced split.**
`WizardViewModel.cs:140` sets `AdvancedFeatures = true` in `Finish()` (both Next-on-last-step and `Skip`).
The split exists so "the page isn't a wall of options for a first-time user" (`SettingsPage.xaml:25-26`).
Since the default launch path *is* the wizard, essentially every new user lands with Advanced ON. The
`AdvancedFeatures=false` default (`ISettingsStore.cs:18`) is effectively dead. *Fix:* don't flip Advanced on
at setup completion.

**5. Voice-rewrite's "use the Stop button" is unreachable.**
`PillController.cs:82` arms no cancel key for voice rewrite ("use the Stop button"). But `StopButton` lives in
the collapsed expand panel (`PillWindow.xaml:54-58`), and the pill only expands when there's transcript text
(`PillWindow.xaml.cs:295`). Voice-rewrite recording produces no partials (`PillController.cs:108-111`), so the
panel never opens. Only exit is re-pressing the undiscoverable Alt+Period. *Fix:* show Stop inline in the
recording row for voice rewrite, or arm Esc as stop there.

## P2 — High-value UX/UI improvements

**6. The voice-rewrite hotkey is active but has no UI.** `App.xaml.cs:2099` registers `RewriteWithVoice`
(default Alt+Period, `ISettingsStore.cs:55`), but `ShortcutsPage.xaml` never lists it. Add a Shortcuts row
(pairs with P1-3).

**7. "Set as default" in the picker is undiscoverable.** Footer legend lists ↑↓/Enter/Ctrl+P/Esc
(`PromptPickerWindow.xaml:98-104`) but omits Ctrl+D, bound at `PromptPickerWindow.xaml.cs:93`. Add
"Ctrl+D default."

**8. Wizard offers 7 languages; the engine/Settings offer 33.** `WizardViewModel.cs:21-22` vs
`SettingsViewModel.cs:41-48`. A user whose language is supported can't pick it at setup. Use the same list.

**9. Wizard implies per-language model downloads.** `WizardViewModel.cs:84` — "downloads the right on-device
model for the language you pick." It's a single multilingual Nemotron model and the wizard downloads nothing.
Reword to "one multilingual model handles every language."

**10. "Show advanced features" undersells what's inside.** `SettingsPage.xaml:145-146` enumerates a subset;
the panel also has Launch at login, Keep audio, Return-to-origin, Live captions, Clean-up transcripts. List
them all or drop the enumeration.

**11. The Prompts management page is unreachable from the shell.** `MainWindow.xaml:37` marks the Prompts nav
item `Visibility="Collapsed"` and nothing reveals it. So "Add your own prompt," edit, delete, pin, set-default
(`PromptsPage.xaml:70-123`) is only hit via `--smoketest`. Built-ins work through the picker, and pin/default
via Ctrl+P/Ctrl+D — but there's no in-product way to create a custom prompt. Not marked "HIDDEN until real"
(unlike its siblings). *Confirm intent; if shipping the 33-prompt library, make Prompts reachable (or at least
link to it from the picker).*

## P3 — Polish / nice-to-haves

**12.** "Recents" nav uses a house icon (`MainWindow.xaml:29` `Home24`). Icon says Home, label says Recents.
**13.** Default-prompt badge inconsistent: picker lowercase "default" text (`PromptPickerWindow.xaml:85`) vs
Prompts page capitalized "Default" pill + lightning icon (`PromptsPage.xaml:64,75-78`). Unify.
**14.** Two primary buttons compete in Donations: `$2` and `$10` both `Appearance="Primary"`
(`DonationsWindow.xaml:79-82`). Make one Secondary.
**15.** Help references a push-to-talk feature that doesn't exist (`HelpPage.xaml:54`); `PushToTalkHotkey`
(`ISettingsStore.cs:52`) is never surfaced. Drop the sentence.
**16.** Startup balloon "Double-click the tray icon to open Jot" always fires (`App.xaml.cs:292-294`) even when
the window is shown immediately. Suppress on normal launch.
**17.** Stale pill hint comment: `PillWindow.xaml:42` example "Alt + Space to stop · Esc to cancel" vs actual
"Alt + Space or Esc to stop" (`PillWindow.xaml.cs:317-328`). Align comments.
**18.** "Test connection" row has no Description (`SettingsPage.xaml:135`) while all AI siblings do.

## Intentionally hidden (leave alone)

- **Ask Jot** nav + page — `MainWindow.xaml:32` Collapsed; on-device docs chat, not wired for release.
- **Vocabulary / custom terms** — `SettingsPage.xaml:244-295` Collapsed, "HIDDEN until real: in-memory only."
- **About → Check for updates** — `AboutPage.xaml:28-31` Collapsed, "HIDDEN until real… Velopack auto-update."
- **Recording detail → Detect speakers** — `RecordingDetailPage.xaml:49-52` Collapsed, diarization stub.
- **PFB / Sony provider** sign-in — compiled out on public/Store flavor (`BuildFlavor.cs:22-33`).

## Top 5 if you only do five things
1. Make the wizard's "model ready" tell the truth — bind to `IsModelInstalled`; reconcile 600 MB / 0.67 GB (P1-1).
2. Fix "Launch at login" copy — it opens the window, not the tray (P1-2).
3. Reconcile Rewrite docs vs behavior + surface the hidden voice-rewrite shortcut (P1-3, P2-6).
4. Stop force-enabling Advanced at wizard completion (P1-4).
5. Make Prompts reachable (or confirm hidden) + add Ctrl+D to the picker legend (P2-11, P2-7).
